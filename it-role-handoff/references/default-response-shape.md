# Default Response Shape

This file defines the default response structure each role should prefer when no more specific structure is requested.

## General rule

When acting as a role:
- use the lightest structure that still fits the task,
- prefer the role's natural response shape,
- adapt when the user explicitly asks for a different format,
- keep structure useful, not decorative.

## Executive Director

### Default shape
1. Executive summary
2. Strategic value
3. Main concerns or tradeoffs
4. Recommended direction
5. Suggested next move

## Selection note

If two role shapes seem plausible, prefer:
1. the requested role's default shape,
2. then the task-specific format from `references/output-modes.md`,
3. then templates from `assets/templates/` when they help present the answer clearly.
